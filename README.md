# Lexy with Chat

## О проекте

Lexy with Chat — это веб-платформа для изучения иностранных языков на основе карточек и интервального повторения. Проект сочетает учебный функционал и реальное общение: пользователи создают личные и публичные колоды, повторяют слова и фразы, а также общаются в чате.

## Основные возможности

- Регистрация и авторизация пользователей через JWT
- Управление собственными колодами карточек
- Создание, редактирование и удаление карточек в колодах
- Публикация колод и поиск публичных коллекций
- Адаптивный режим обучения с выбором ответов
- Система избранных карточек и отдельная коллекция избранного
- Статистика обучения: серия занятий, активность и точность ответов
- Профиль пользователя с управлением данными и настройками
- Администрирование: управление ролями, блокировка пользователей, проверка заявок на публикацию
- Реальный чат между пользователями на Socket.IO
- Отдельный сервер чата с шифрованием сообщений

## Архитектура проекта

Проект разделен на несколько компонентов:

- `frontend` — клиентская часть на React и Vite
- `server` — основной API-сервер на Node.js и Express
- `chat-server` — сервер реального времени для чата на Socket.IO
- `postgres` — база данных PostgreSQL с двумя базами: `lexy` и `chat`

## Использованные технологии

- **Frontend:** React 19 + Vite
- **Backend:** Node.js + Express
- **Реальное время:** Socket.IO
- **База данных:** PostgreSQL
- **Контейнеризация:** Docker и Docker Compose
- **Аутентификация:** JWT

## Запуск через Docker Compose

1. Скачайте репозиторий или клонируйте его:

```bash
git clone <https://github.com/karma-ofc/Lexy_With_Chat>
cd Lexy_With_Chat-main
```

Если репозиторий загружен в виде ZIP, распакуйте его и перейдите в папку проекта.

2. Запустите контейнеры:

```bash
docker compose up --build
```

3. Дождитесь старта всех сервисов.

### Доступные адреса

- Фронтенд: `http://localhost:5173`
- Основной API: `http://localhost:3000`
- Сервер чата: `http://localhost:3001`
- PostgreSQL: `localhost:5433`

## Сервисы в Docker Compose

- `frontend` — запускает клиентское приложение
- `main-server` — основной сервер API
- `chat-server` — сервер чата
- `postgres` — база данных PostgreSQL

## Переменные окружения

Проект использует переменные окружения, которые можно настраивать через `.env` или через `docker-compose.yml`.

Основные параметры:

- `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`
- `MAIN_DB_NAME` — основная база данных
- `CHAT_DB_NAME` — база данных чата
- `MAIN_CHAT_API_URL` — URL сервера чата для основного API
- `JWT_SECRET` — секрет для JWT
- `ENCRYPTION_KEY` — ключ шифрования для чата
- `YANDEX_DICT_API_KEY` — ключ для переводчика от яндекса
- `VAPID_PUBLIC_KEY` и `VAPID_PRIVATE_KEY` — ключи для веб-пушей

## Структура репозитория

- `src/` — клиентская часть приложения
- `server/` — код основного API
- `chat-server/` — код сервера чата
- `docker-compose.yml` — описание Docker-сервисов
- `Dockerfile` — образ для сервера
- `Dockerfile.frontend` — образ для фронтенда
- `migrations/` — утилиты и миграции БД

## Дополнительно

- В проекте присутствуют разделы, связанные с администрированием и публикацией колод
- Чат работает как отдельный сервис и подключается к основному приложению через сетевой API
