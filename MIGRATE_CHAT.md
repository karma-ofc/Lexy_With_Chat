# Миграция данных чата: Lexy → Chat DB

## Цель

Перенести все таблицы, связанные с микросервисом чата, из основной базы `lexy` в отдельную базу `chat`.

## ✨ Подход: Автоматическая миграция при старте (текущая версия)

**Начиная с текущей версии, миграция происходит автоматически!**

1. Chat-сервер при старте проверяет, нужна ли миграция (если в source есть пользователи, а в target нет)
2. Если нужна → автоматически выполняет миграцию
3. Затем инициализирует БД и запускается нормально

### Просто запустите:

```bash
npm run start
```

Миграция произойдёт автоматически, если это требуется. Вы увидите в логах:

```
[Migration] Starting automatic chat data migration from lexy to chat database...
[Migration] Found X roles, Y users, Z groups, W messages
...
[Migration] ✓ Chat migration completed successfully!
```

### Если нужна ручная миграция

Если по какой-то причине вам нужно выполнить миграцию вручную (например, после изменения .env):

```bash
npm run migrate:chat
```

Это полезно, если вы хотите пересинхронизировать данные или выполнить миграцию в другое время.

## Что переносится

- roles
- users
- chat_groups
- chat_messages

## Важные детали

- Пользователи (`users`) копируются в `chat` с сохранением `id`, чтобы чат-сервис и сообщения сохранили ссылочную целостность
- Перед выполнением убедитесь, что у вас есть доступ к обеим БД и корректные `.env` в `server/` и `chat-server/`
- Миграция создаёт резервные копии в `migrations/backups/` перед переносом данных
- После миграции выполняется проверка целостности данных (referential integrity check)

## Резервные копии

Все резервные копии сохраняются в `migrations/backups/` с меткой времени:
- `source-before.json` — состояние source DB перед миграцией
- `target-before.json` — состояние target DB перед миграцией

## Логика автоматической миграции

Миграция пропускается, если:
- В target БД уже есть пользователи (в этом случае данные уже синхронизированы)
- Source БД не содержит пользователей

Миграция выполняется только при условии:
- Source БД содержит пользователей И
- Target БД пуста

## Если нужна помощь

Для уточнения деталей миграции или если возникли проблемы, проверьте логи с префиксом `[Migration]`.
